# Project Rules & Guidelines

## Frameworks and Libraries
- **React**: Use functional components with hooks.
- **TypeScript**: Ensure strict type checking. Use interfaces for component props.
- **Tailwind CSS**: Use Tailwind CSS v3 for styling. Prefer utility classes over custom CSS.
- **Vite**: Use Vite as the build tool and development server.
- **Lucide React**: Use for icons.

## Frontend Guidelines
- **Component Structure**: Follow the atomic design principles. Reusable UI components in `src/components/ui/`.
- **Styling**: Use Tailwind CSS for all styling. Maintain a consistent color palette and spacing.
- **Responsive Design**: Ensure all components and pages are mobile-friendly using Tailwind responsive prefixes (`sm:`, `md:`, `lg:`, etc.).
- **Performance**: Optimize images and use lazy loading where appropriate.
- **Accessibility**: Use semantic HTML elements and ensure proper ARIA attributes.

## Agent Behavior
- **Vietnamese Language**: Always communicate in Vietnamese as per user preference.
- **Code Quality**: Write clean, documented, and maintainable code.
- **Problem Solving**: Proactively identify and fix issues (e.g., missing dependencies, configuration errors).
- **Communication**: Be concise and provide clear summaries of actions taken.
- **User Confirmation**: Seek approval for major changes but handle minor fixes autonomously.
